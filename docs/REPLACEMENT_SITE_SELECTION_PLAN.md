# 대체 사이트 선정 계획서

## 개요
리스크가 높은 4개 사이트의 크롤링을 중단한 후, 대체 가능한 체험단 플랫폼을 체계적으로 선정하기 위한 계획서입니다.

**작성일**: 2025년 1월  
**목적**: 대체 사이트 선정 기준 수립 및 우선순위 결정

---

## 선정 기준

### 1. 법적/기술적 리스크 (필수 조건)

#### 1.1 robots.txt 준수
- ✅ **크롤링 가능**: robots.txt에서 일반 경로 허용
- ⚠️ **부분 제한**: 일부 경로만 금지 (목록 페이지는 가능)
- ❌ **크롤링 불가**: 필수 경로 금지 또는 전체 금지

#### 1.2 이용약관 준수
- 자동화된 데이터 수집 금지 조항 확인 필요
- 상업적 사용 제한 확인 필요

**평가 기준**:
- 🟢 **낮음**: robots.txt 허용 + 이용약관 확인 필요
- 🟡 **중간**: robots.txt 부분 제한 또는 이용약관 확인 필요
- 🔴 **높음**: robots.txt 위반 또는 이용약관 위반

---

### 2. 기술적 크롤링 가능성

#### 2.1 사이트 구조
- **정적 HTML**: 크롤링 용이
- **동적 로딩 (JavaScript)**: Selenium/Playwright 필요
- **API 기반**: API 엔드포인트 존재 여부

#### 2.2 URL 패턴
- **규칙적 패턴**: `/campaign/{id}`, `/list?page={n}` 등
- **불규칙 패턴**: 해시 기반 또는 복잡한 쿼리 파라미터

#### 2.3 인증 요구사항
- **공개 접근**: 로그인 불필요
- **제한적 접근**: 일부 기능만 로그인 필요
- **로그인 필수**: 전체 로그인 필요

**평가 기준**:
- 🟢 **쉬움**: 정적 HTML + 규칙적 URL + 공개 접근
- 🟡 **보통**: 동적 로딩 또는 부분 인증 필요
- 🔴 **어려움**: 복잡한 인증 또는 API 키 필요

---

### 3. 데이터 품질

#### 3.1 데이터 완전성
- **필수 필드**: 제목, URL, 카테고리, 마감일, 지역, 이미지
- **추가 필드**: 채널, 리뷰 기간, 보상, 모집 인원

#### 3.2 데이터 일관성
- 구조화된 데이터 형식
- 표준화된 카테고리 체계
- 일관된 날짜 형식

#### 3.3 데이터 양
- 일일 신규 캠페인 수
- 전체 활성 캠페인 수
- 업데이트 빈도

**평가 기준**:
- 🟢 **우수**: 필수 필드 모두 + 구조화 + 일일 50개 이상
- 🟡 **보통**: 필수 필드 대부분 + 부분 구조화 + 일일 20-50개
- 🔴 **부족**: 필수 필드 누락 + 비구조화 + 일일 20개 미만

---

### 4. 사이트 안정성

#### 4.1 운영 상태
- 정기적인 업데이트
- 안정적인 서비스 운영
- 장기 운영 가능성

#### 4.2 기술적 안정성
- 서버 다운 빈도
- 응답 속도
- 에러 발생률

**평가 기준**:
- 🟢 **안정적**: 정기 업데이트 + 낮은 다운타임
- 🟡 **보통**: 간헐적 업데이트 + 가끔 다운타임
- 🔴 **불안정**: 업데이트 없음 + 빈번한 다운타임

---

### 5. 비즈니스 적합성

#### 5.1 타겟 사용자
- cally.kr 사용자와 유사한 타겟
- 체험단 신청자 풀

#### 5.2 카테고리 다양성
- 맛집, 뷰티, 배송, 문화 등 다양한 카테고리
- 지역별 분포

**평가 기준**:
- 🟢 **높음**: 다양한 카테고리 + 전국 지역
- 🟡 **보통**: 주요 카테고리 + 주요 지역
- 🔴 **낮음**: 제한적 카테고리 + 특정 지역

## 선정 프로세스

### Step 1: 사전 검증 (1일)

1. **모두의체험단 동일 사이트 확인**
   - `modan.kr`과 `xn--6j1br0ag3lba435lvsj96p.com`이 동일 사이트인지 확인
   - 확인 방법:
     - DNS 조회
     - 사이트 구조 비교
     - 운영사 정보 확인

2. **이용약관 확인**
   - 각 사이트의 이용약관 전문 확인
   - 자동화된 데이터 수집 관련 조항 확인
   - 상업적 사용 제한 확인

### Step 2: 기술적 검증 (2-3일)

1. **사이트 구조 분석**
   - HTML 구조 확인
   - JavaScript 의존성 확인
   - API 엔드포인트 확인

2. **크롤링 테스트**
   - 목록 페이지 크롤링 테스트
   - 데이터 추출 테스트
   - 에러 처리 테스트

### Step 3: 데이터 품질 검증 (1일)

1. **샘플 데이터 수집**
   - 각 사이트에서 10-20개 샘플 수집
   - 필수 필드 완전성 확인
   - 데이터 형식 일관성 확인

2. **데이터 양 확인**
   - 일일 신규 캠페인 수 확인
   - 전체 활성 캠페인 수 확인

### Step 4: 최종 선정 (1일)
